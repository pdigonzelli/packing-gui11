Application Manager: Application Config

Application Config is unique to each broker. It let you configure which code directories you want to deal with.

1. Select directories containing your Progress source-code.

  • It lists all propath components that are not Progress Install Source ($DLC).
  • It will let you recurse up to 3 levels on each of the propath components if you wish so, by pressing the 1, 2 or 3 level buttons.
  • Additionally you can add directories that don't appear as propath components by clicking "Add unlisted" and specify it directly.

  • Simply select the codepaths you want to add to the main menu and Click the "OK" button.

 

2. Configure broker for RCS.

If you do not wish to use the built-in RCS for task managment then DO NOT PRESS THIS BUTTON.

  • This broker will now be the code repository broker. It will place a RCS/rcs.ini file in its current directory to identify it as the repository broker. Any other broker with this path in its propath will be considered users, meaning that a checkout process will place the files in the "users" current directory for when checking out.
  • It uses the standard /RCS subdirectory (Unix way) to store the repository files.
